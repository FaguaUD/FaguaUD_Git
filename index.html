FaguaUD

FAGUA-UD es un producto que permite la especificación y documentación de los requerimientos de negocio asociados al desarrollo de un proyecto de Data Mart.

View project on GitHub

Funcionalidades:

  • Los formularios dinámicos permiten la creación de una comunidad en torno a un repositorio. Por medio de esta comunidad se proyecta el constante mantenimiento y mejora de los formularios, así mismo, su especialización en diferentes campos de negocio de acuerdo a la experticia y el aporte de las personas que integren la comunidad.

  • Los roles definidos abarcan todo el ciclo de vida de la etapa de requerimientos de un proyecto de Data mart. Esta definición de roles permite un uso de la herramienta adaptable a equipos de trabajo de diferentes tamaños.

  • La herramienta Fagua-UD se desarrolló en un entorno web el cual permite acceso permanente en línea y a través de diferentes dispositivos (Equipos de Escritorio, Equipos Portátiles, Tabletas y Smartphones principalmente).

  • Con base en el prototipo grafico del cubo se planteó el desarrollo e implementación de una funcionalidad adicional que permite a la herramienta Fagua-UD la generación de scripts. Estos scripts permiten la creación de los objetos del modelo estrella para diferentes motores de bases de datos (Oracle, Postgress y SQLServer).

Beneficios:

  • Debido a la importancia de la etapa de levantamiento de requerimientos, la Herramienta Fagua-UD sirve como una base sólida a la hora de emprender dicha etapa en un proyecto de Data mart.

  • La utilización de la Herramienta Fagua-UD permite trabajar de manera organizada a la hora de realizar el levantamiento de requerimientos de un proyecto de Data mart, dejando definidas las jerarquías de mando y a su vez permitiendo la trazabilidad de todas las etapas del proceso de acuerdo con el esquema metodológico propuesta por el autor Ralph Kimball.

  • Para la Herramienta Fagua-UD se definieron formularios dinámicos con el propósito de estandarizar y agilizar el proceso de levantamiento de requerimientos de un proyecto de Data mart. Esto mejora la adaptabilidad y escalabilidad del proyecto sin limitar el entorno de negocio al cual se pueda aplicar.

  • La Herramienta Fagua-UD permite visualizar mediante un modelo gráfico estrella un prototipo inicial del cubo, una tabla de hechos y las dimensiones que lo acompañan, incluyendo sus atributos asociados.

  • La Herramienta Fagua-UD se presenta como una primera versión enfocada en la fase de requerimientos de un proyecto de Data mart. Sin embargo, se podrían generar nuevas versiones que contemplen fases posteriores del proceso (Diseño arquitectura, Modelamiento dimensional, Diseño físico, despliegue, etc.).
